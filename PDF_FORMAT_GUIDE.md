# Руководство по формату PDF файлов для Go2Study Bot

## Новый поддерживаемый формат (обновлено 2024-12-19)

PDF процессор теперь поддерживает упрощенный и стандартизированный формат:

### **Основной формат (рекомендуемый)**
```
Тема: Пропорция(10)

1) Найдите значение x в пропорции 2:3 = x:12
A) 6
B) 8 ✅
C) 9
D) 10

2) Решите пропорцию 5:x = 15:9
A) 3 ✅
B) 4
C) 5
D) 6

...

10) Последний вопрос по теме
A) Вариант A
B) Вариант B ✅
C) Вариант C
D) Вариант D

Тема: Уравнение(5)

1) Решите уравнение: 2x + 5 = 13
A) x = 3
B) x = 4 ✅
C) x = 5
D) x = 6

...
```

## Требования к формату

### **Обязательные элементы:**

1. **Заголовок темы**: `Тема: [Название темы]([Количество вопросов])`
   - Пример: `Тема: Пропорция(10)`
   - Пример: `Тема: Линейные уравнения(15)`

2. **Нумерация вопросов**: `1)`, `2)`, `3)`, ... (с закрывающей скобкой)

3. **Варианты ответов**: 
   - Латинские: `A)`, `B)`, `C)`, `D)`
   - Или кириллические: `А)`, `Б)`, `В)`, `Г)`

4. **Маркер правильного ответа**: `✅` (галочка после текста ответа)
   - Пример: `B) 8 ✅`

### **Структура документа:**
- Каждая тема начинается с заголовка
- Вопросы нумеруются с 1 для каждой новой темы
- Пустые строки между элементами для лучшей читаемости
- Один правильный ответ на вопрос

## Автоматическая нормализация тем

Процессор автоматически сопоставляет названия тем из PDF с темами из системы:

### Поддерживаемые соответствия:
- `Пропорция` → `Соотношение и пропорция`
- `Уравнение/Уравнения` → `Линейные уравнения`
- `Дроби/Дробь` → `Операции с дробями и остатками`
- `Процент/Проценты` → `Процент`
- `Геометрия` → `Геометрия`
- `Текстовые задачи` → `Текстовые задачи`
- `Логика` → `Логические задачи`
- `Функция/График` → `График функции`
- `Модуль` → `Модуль числа`
- И другие...

Если точное соответствие не найдено, система использует AI для определения наиболее подходящей темы.

## Языковая поддержка

- **Русский язык**: полная поддержка
- **Казахский язык**: автоматическое определение по символам әіңғүұқөһ
- **Смешанный контент**: поддерживается

## Технические требования

- **Кодировка**: UTF-8
- **Формат файла**: PDF (любая версия)
- **Символы**: поддерживаются математические символы, дроби, проценты
- **Максимальная длина вопроса**: без ограничений
- **Максимальная длина ответа**: 200 символов

## Процесс обработки

1. **Обнаружение тем**: поиск заголовков `Тема: название(количество)`
2. **Извлечение вопросов**: поиск нумерованных вопросов `1)`, `2)`, etc.
3. **Парсинг ответов**: извлечение вариантов A), B), C), D)
4. **Определение правильного ответа**: поиск символа ✅
5. **Нормализация темы**: сопоставление с системными темами
6. **Валидация**: проверка полноты данных
7. **Сохранение в базу**: добавление уникальных вопросов

## Устранение проблем

### Если вопросы не обрабатываются:
1. Проверьте формат заголовков тем
2. Убедитесь в правильной нумерации вопросов
3. Проверьте наличие символов ✅ для правильных ответов
4. Убедитесь в правильном формате вариантов ответов

### Если темы определяются неправильно:
1. Используйте точные названия из списка поддерживаемых тем
2. Проверьте правильность написания в заголовке
3. Система автоматически попытается найти наиболее подходящую тему

## Добавление нового PDF файла

1. Поместите PDF файл в папку `files/`
2. Откройте `src/services/pdf_processor.py`
3. Добавьте путь к файлу в список `pdf_files` в функции `main()`
4. Запустите: `python -m src.services.pdf_processor`

## Логирование и отладка

Процессор предоставляет подробные логи:
- Обнаруженные темы и количество вопросов
- Прогресс обработки каждого вопроса
- Найденные правильные ответы
- Статистику сохранения в базу данных
- Ошибки валидации

Все добавленные вопросы записываются в файл `added_questions.log`.
