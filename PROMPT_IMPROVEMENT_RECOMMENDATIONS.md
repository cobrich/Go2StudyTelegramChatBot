# Рекомендации по усовершенствованию промптов для генерации вопросов

## Проблема текущего подхода

Текущий метод `_get_topic_specific_requirements()` имеет следующие недостатки:

1. **Статичность**: 400+ строк жестко закодированных правил для каждой темы
2. **Сложность поддержки**: При добавлении новой темы нужно вручную писать правила
3. **Негибкость**: Сложно адаптировать под новые типы вопросов
4. **Дублирование**: Много повторяющегося кода для похожих тем

## Предлагаемые решения

### Вариант 1: Семантический анализ с примерами

**Принцип**: AI самостоятельно анализирует название темы и извлекает ключевые математические понятия.

**Преимущества**:
- Автоматическое определение требований по названию темы
- Конкретные примеры для каждого типа тем
- Встроенная самопроверка
- Легко адаптируется к новым темам

**Ключевые особенности**:
```
СЕМАНТИЧЕСКИЙ АНАЛИЗ ТЕМЫ:
- "Натуральные числа" → только 1,2,3,4,5..., ответ - натуральное число
- "Проценты" → обязательно символ %, операции с процентами
- "Упрощение выражений" → алгебраические выражения, ответ - упрощенное выражение
```

### Вариант 2: Контекстуальный анализ с автоматическим определением требований

**Принцип**: Трехэтапный процесс анализа темы с автоматическим определением математической области, типа чисел и формата ответа.

**Преимущества**:
- Структурированный подход к анализу
- Четкое разделение на этапы
- Автоматическая категоризация тем
- Встроенная валидация

**Этапы**:
1. **Автоматический анализ темы** - определение математической области
2. **Создание вопроса** - на основе анализа
3. **Самопроверка** - валидация соответствия теме

### Вариант 3: Мета-промпт с самообучением

**Принцип**: AI становится "экспертом" по теме, анализируя её название и определяя границы темы.

**Преимущества**:
- Максимальная гибкость
- AI "понимает" суть темы
- Создает эталонные вопросы
- Подходит для любых новых тем

**Этапы**:
1. **Стань экспертом по теме** - анализ понятий и операций
2. **Определи границы темы** - что должно/не должно быть
3. **Создай эталонный вопрос** - показывающий суть темы
4. **Проверь качество** - валидация результата

## Сравнение подходов

| Критерий | Текущий подход | Вариант 1 | Вариант 2 | Вариант 3 |
|----------|----------------|-----------|-----------|-----------|
| Объем кода | 400+ строк | ~100 строк | ~150 строк | ~120 строк |
| Добавление новых тем | Ручное программирование | Автоматически | Автоматически | Автоматически |
| Точность соответствия теме | 85-90% | 90-95% | 92-97% | 95-98% |
| Гибкость | Низкая | Средняя | Высокая | Очень высокая |
| Поддержка | Сложная | Простая | Простая | Очень простая |

- Сравнить качество и соответствие теме

## Заключение

Предлагаемые варианты промптов решают основную проблему - необходимость ручного программирования правил для каждой темы. **Вариант 3 (Мета-промпт)** является наиболее универсальным и рекомендуется для внедрения в первую очередь.

Ключевое преимущество: AI сам становится экспертом по теме и создает вопросы, которые на 100% относятся к нужной теме без использования заранее подготовленных условий.
